# Notifications Design Spec

**Date:** 2026-03-24
**Branch:** `feature/notifications`
**Status:** Approved

---

## Overview

Implement a dual-channel notification system for BellyBuddy: **local notifications** for time-based reminders and **push notifications** (via Firebase Cloud Messaging) for server-triggered alerts.

---

## Notification Types

| # | Type | Channel | Trigger | Deep Link | Default |
|---|------|---------|---------|-----------|---------|
| 1 | Logging reminder | Local | User-configured times (HH:mm) | `/dashboard` | 18:00 |
| 2 | Daily gut summary | Local | User-configured time (HH:mm) | `/gut-feeling-tracker` | 20:00 |
| 3 | New recommendation | Push | DB trigger on `recommendations` INSERT | `/recommendations` | — |
| 4 | Ingredient suggestion | Push | DB trigger on `ingredient_suggestions` INSERT | `/ingredient-suggestions` | — |
| 5 | Inactivity nudge | Push | Cron (daily), 3 days no entries, fires once | `/dashboard` | — |

## Data Model Changes

### New Fields on `UserProfile`

| Field | Type | Default | DB Column | Purpose |
|-------|------|---------|-----------|---------|
| `remindersEnabled` | `bool` | `true` | `reminders_enabled` | Toggle for local logging reminders |
| `dailySummaryEnabled` | `bool` | `true` | `daily_summary_enabled` | Toggle for daily gut feeling reminder |
| `pushEnabled` | `bool` | `false` | `push_enabled` | Master toggle for push notifications |
| `dailySummaryTime` | `String` | `"20:00"` | `daily_summary_time` | HH:mm for gut feeling reminder |
| `fcmToken` | `String?` | `null` | `fcm_token` | Device FCM token for push delivery |
| `lastInactivityNudge` | `DateTime?` | `null` | `last_inactivity_nudge` | Prevents repeat nudges |

### Modified Fields

| Field | Before | After | Migration |
|-------|--------|-------|-----------|
| `reminderTimes` | `List<int>` (hours) | `List<String>` (HH:mm) | `18` → `"18:00"` |

### Supabase Migration

Add 6 columns to `profiles` table:
- `reminders_enabled BOOLEAN DEFAULT true`
- `daily_summary_enabled BOOLEAN DEFAULT true`
- `push_enabled BOOLEAN DEFAULT false`
- `daily_summary_time TEXT DEFAULT '20:00'`
- `fcm_token TEXT`
- `last_inactivity_nudge TIMESTAMPTZ`

Migrate existing `reminder_times`: convert each integer to `"HH:00"` string format.

```sql
-- Convert reminder_times from int array to string array
UPDATE profiles
SET reminder_times = (
  SELECT array_to_json(array_agg(lpad(elem::text, 2, '0') || ':00'))
  FROM jsonb_array_elements_text(reminder_times::jsonb) AS elem
)
WHERE reminder_times IS NOT NULL;
```

**Dart-side migration note:** During the rollout window, the Freezed model should use a custom `JsonConverter` on `reminderTimes` that accepts both `int` and `String` values, converting ints to `"HH:00"` format. This ensures the app handles both old and new data formats gracefully. After all clients have updated, the converter can be simplified.

**Build runner:** After model changes, run `dart run build_runner build --delete-conflicting-outputs` to regenerate `.freezed.dart` and `.g.dart` files.

## Local Notification Scheduling

### `LocalNotificationService`

**Logging reminders:**
- For each time in `reminderTimes`, schedule a daily repeating notification
- Android channel: `logging_reminders` ("Erinnerungen")
- Timezone-aware using `timezone` package + user's `timezone` field
- Body randomly selected from a hardcoded list in `LocalNotificationService`:
  - "Zeit zum Eintragen! Was hast du gegessen?"
  - "Vergiss nicht, deine Mahlzeiten zu tracken!"
  - "Wie geht es deinem Bauch? Trag es ein!"
  - "Erinnerung: Halte dein Essens-Tagebuch aktuell."
- Tap → `/dashboard`

**Daily gut summary:**
- Single daily repeating notification at `dailySummaryTime`
- Android channel: `daily_summary` ("Tägliche Zusammenfassung")
- Body: "Wie war dein Bauchgefühl heute?"
- Tap → `/gut-feeling-tracker`

**Android notification channels:**
- Created during `LocalNotificationService` initialization
- `logging_reminders`: name "Erinnerungen", description "Tägliche Erinnerungen zum Tracken"
- `daily_summary`: name "Tägliche Zusammenfassung", description "Abendliche Bauchgefühl-Erinnerung"

**Scheduling lifecycle:**
1. App start → schedule all (idempotent: cancel existing, reschedule)
2. Settings change → cancel + reschedule affected type
3. Sign out → cancel all

### Notification IDs

- Logging reminders: IDs `1000–1099` (one per configured time, max 100)
- Daily summary: ID `2000`

## Push Notification Flow

### FCM Token Lifecycle

1. App start → `firebase_messaging` requests permission
2. Permission granted → get FCM token, save to `profiles.fcm_token`
3. Token refresh → update `profiles.fcm_token`
4. Push toggle off → set `push_enabled = false` (keep token for re-enabling)
5. Sign out → clear `fcm_token` from profile

### Message Handling

**Foreground:** `PushNotificationService` exposes a `Stream<RemoteMessage>` that the app shell (`app.dart`) listens to via a `StreamProvider`. When a message arrives, show a `SnackBar` via a `GlobalKey<ScaffoldMessengerState>` or an overlay. Tapping the snackbar navigates to `data.route`.

**Background/terminated:** FCM displays notification natively. On tap → app opens, `PushNotificationService` reads `data.route` from the initial message. Navigation uses a `GlobalKey<NavigatorState>` passed to `GoRouter` and shared with `PushNotificationService`, enabling navigation without `BuildContext`.

### Payload Structure

```json
{
  "notification": {
    "title": "Neue Empfehlung",
    "body": "Du hast eine neue Ernährungsempfehlung!"
  },
  "data": {
    "route": "/recommendations"
  }
}
```

## Supabase Edge Functions

### `send-push-notification` (update existing)

- Accepts `{ userId, title, body, route }` or `{ fcmToken, title, body, route }`
- If `userId` provided: look up `fcm_token` and `push_enabled` from `profiles`
- Sends via FCM HTTP v1 API using Firebase service account key (Supabase secret: `FIREBASE_SERVICE_ACCOUNT`)
- Returns success/error

### `check-inactivity` (new, cron-triggered daily)

- Runs daily (e.g., 10:00 UTC)
- Query: users where `push_enabled = true` AND no entry in any tracking table for 3+ days AND (`last_inactivity_nudge` is null OR older than the last entry)
- Send push: title "Wir vermissen dich!", body "Du hast seit 3 Tagen nichts eingetragen.", route `/dashboard`
- Update `last_inactivity_nudge` to now

### Database Triggers

Implemented as **Supabase Database Webhooks** (not Postgres triggers), configured in the Supabase Dashboard:

- `recommendations` INSERT → calls `send-push-notification` edge function with `{ userId, title, body, route: "/recommendations" }`
- `ingredient_suggestions` INSERT → calls `send-push-notification` edge function with `{ userId, title, body, route: "/ingredient-suggestions" }`

The edge function checks `push_enabled = true` before sending.

### Edge Function Management

Edge functions are managed via the **Supabase Dashboard** (not version-controlled in this repo). The spec includes the full function code for reference, to be pasted into the Dashboard's edge function editor. The API contract between the Flutter app and the edge functions is what matters for the client-side implementation.

## Firebase Setup Guide

### 1. Firebase Console

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Create a new project (or select existing)
3. **Add iOS app:**
   - Bundle ID: find in Xcode → Runner → General → Bundle Identifier
   - Download `GoogleService-Info.plist`
   - Place in `ios/Runner/GoogleService-Info.plist`
4. **Add Android app:**
   - Package name: find in `android/app/src/main/AndroidManifest.xml` → `package` attribute
   - Download `google-services.json`
   - Place in `android/app/google-services.json`
5. **Enable Cloud Messaging:**
   - Project Settings → Cloud Messaging → ensure it's enabled
6. **Generate Firebase options:**
   - Install FlutterFire CLI: `dart pub global activate flutterfire_cli`
   - Run `flutterfire configure` in the project root
   - Select your Firebase project and platforms (iOS + Android)
   - This generates `lib/firebase_options.dart` (required for `Firebase.initializeApp()`)

### 2. iOS Configuration

**Apple Developer Portal:**
1. Go to [Apple Developer → Keys](https://developer.apple.com/account/resources/authkeys/list)
2. Create a new key → enable "Apple Push Notifications service (APNs)"
3. Download the `.p8` file — save it securely (you can only download once)
4. Note the **Key ID** and your **Team ID** (from Membership page)

**Upload APNs Key to Firebase:**
1. Firebase Console → Project Settings → Cloud Messaging
2. Under "Apple app configuration" → click the upload button
3. Upload the `.p8` file, enter Key ID and Team ID

**Xcode:**
1. Open `ios/Runner.xcworkspace` in Xcode
2. Select Runner → Signing & Capabilities
3. Click "+ Capability" → add **Push Notifications**
4. Click "+ Capability" → add **Background Modes** → check **Remote notifications**

### 3. Android Configuration

**`android/app/build.gradle`:**
- Add `apply plugin: 'com.google.gms.google-services'` at the bottom

**`android/build.gradle` (project-level):**
- Add `classpath 'com.google.gms:google-services:4.4.2'` to `dependencies`

**`android/app/src/main/AndroidManifest.xml`:**
- Add `<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />`
- Add `<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />`
- Add `<uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM" />` (required for Android 14+ exact alarm scheduling)

### 4. Supabase Secret for Server-Sent Push

1. Firebase Console → Project Settings → Service Accounts
2. Click "Generate new private key" → downloads a JSON file
3. In Supabase Dashboard → Project Settings → Edge Functions → Secrets
4. Add secret `FIREBASE_SERVICE_ACCOUNT` with the full JSON content
5. Edge functions access it via `Deno.env.get('FIREBASE_SERVICE_ACCOUNT')`

## Error Handling & Edge Cases

- **Permission denied:** Show snackbar with instructions to enable in system settings. Toggle stays off.
- **FCM token failure:** Retry on next app start. Local reminders still work independently.
- **Timezone change:** Cancel and reschedule all local notifications with new timezone.
- **Device restart:** Android: `RECEIVE_BOOT_COMPLETED` restores local notifications. iOS: repeating notifications persist automatically.
- **Sign out:** Clear FCM token from profile, cancel all local notifications.
- **Multiple devices:** Last-registered device receives pushes. Multi-device support deferred.
